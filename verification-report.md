## Methodology Notes

1. **Web Search Strategy:** Used multiple query patterns per resource:
   - "Dartmouth [resource name]"
   - "site:[relevant domain] [resource name]"
   - "[resource type] Dartmouth students"

2. **Verification:** Fetched critical URLs via WebFetch to confirm page content matches resource description

3. **URL Updates:** All Article IDs in services.dartmouth.edu knowledge base were outdated; found current IDs via site search

4. **Consolidation Logic:** Combined resources that:
   - Share same access page (e.g., all news via library page)
   - Are part of same system (e.g., all databases in A-Z list)
   - Would confuse users if listed separately

5. **Name Corrections:** Updated to current official names where services were renamed
